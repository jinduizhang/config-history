# Constitution - 项目开发原则

## 项目概述

- **项目名称**: 配置历史管理模块
- **项目类型**: Spring Boot后端模块 + 管理前端
- **核心功能**: 配置项的增删改查、历史记录查看、版本对比与回退

---

## 技术栈

- **后端**: Spring Boot 3.x + MyBatis-Plus
- **数据库**: MySQL 8.x
- **前端**: Vue 3 + Element Plus（可选）
- **API风格**: RESTful API

---

## 开发原则

### 1. 代码质量

- 遵循阿里巴巴Java开发规范
- Controller层仅做参数校验和响应封装，业务逻辑下沉到Service层
- 所有API必须提供OpenAPI/Swagger文档注解
- 单元测试覆盖率需达到核心业务逻辑70%以上

### 2. 数据库设计

- 表名使用下划线命名法（snake_case）
- 必须包含创建时间、更新时间字段
- 索引命名规范：`idx_字段名`、`uk_字段名`
- 禁止使用物理外键，使用逻辑外键

### 3. API设计

- 遵循RESTful规范，使用标准HTTP方法
- 响应格式统一：`{ code: 200, data: {}, message: "success" }`
- 分页查询使用 `page`, `pageSize` 参数
- 所有写操作需记录操作人（operator）

### 4. 配置历史设计

- 每次配置变更必须生成新版本号（自增）
- 历史记录必须包含：变更内容、变更时间、操作人、变更原因
- 版本回退采用"创建新版本"而非"覆盖"，确保审计完整性
- 支持配置项的导入导出（JSON格式）

### 5. 性能要求

- 单表查询响应时间 < 100ms
- 历史记录查询支持分页
- 大JSON字段需评估索引策略

### 6. 安全要求

- 操作日志需记录操作人IP
- 敏感配置需考虑加密存储
- API需支持权限控制（预留扩展）

- [ ] API文档完整
