# Constitution - 项目开发原则

## 项目概述

- **项目名称**: 通用历史记录管理模块 (Generic History Audit Module)
- **项目类型**: Spring Boot后端模块
- **核心功能**: 任意实体的变更历史记录、版本对比、版本回退
- **设计目标**: 可复用、可扩展、低侵入

---

## 技术栈

- **后端**: Spring Boot 3.x + MyBatis-Plus 3.5.x
- **数据库**: MySQL 8.x / H2 (开发)
- **API风格**: RESTful API
- **构建工具**: Maven

---

## 开发原则

### 1. 代码质量

- 遵循阿里巴巴Java开发规范
- Controller层仅做参数校验和响应封装，业务逻辑下沉到Service层
- 所有API必须提供OpenAPI/Swagger文档注解
- 单元测试覆盖率需达到核心业务逻辑70%以上

### 2. 数据库设计

- 表名使用下划线命名法（snake_case）
- 必须包含创建时间、更新时间字段
- 索引命名规范：`idx_字段名`、`uk_字段名`
- 禁止使用物理外键，使用逻辑外键

### 3. API设计

- 遵循RESTful规范，使用标准HTTP方法
- 响应格式统一：`{ code: 200, data: {}, message: "success", timestamp: xxx }`
- 分页查询使用 `page`, `pageSize` 参数
- 所有写操作需记录操作人（operator）
- API版本控制：`/api/v1/xxx`

### 4. 通用历史记录设计原则

- **泛型设计**: 使用泛型支持任意实体类型
- **注解驱动**: 通过注解标记需要记录历史的实体和字段
- **低侵入性**: 业务代码无需感知历史记录逻辑（AOP切面）
- **快照存储**: 每次变更保存完整快照，确保审计完整性
- **版本号策略**: 版本号自增，回退创建新版本而非覆盖

### 5. 性能要求

- 单表查询响应时间 < 100ms
- 历史记录查询支持分页
- 大JSON字段需评估索引策略
- 支持异步记录历史（可选）

### 6. 安全要求

- 操作日志需记录操作人IP
- 敏感配置需考虑加密存储
- API需支持权限控制（预留扩展）

### 7. 异常处理

- 全局异常处理器统一处理
- 自定义业务异常
- 异常响应格式统一

### 8. 架构设计

- 分层架构：Controller → Service → Mapper
- DTO模式：请求/响应与实体分离
- 接口分离：Service接口与实现分离
- 依赖注入：使用构造器注入
