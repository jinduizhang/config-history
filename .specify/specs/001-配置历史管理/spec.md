# Spec - 配置历史管理功能规范

## 1. 概述

开发配置历史管理模块，支持配置项的变更历史记录、版本对比与回退功能，并提供RESTful API供前台系统调用。

## 2. 用户故事

| 角色 | 用户故事 |
|------|----------|
| 运维人员 | 作为运维人员，我希望查看配置的历史变更记录，以便追踪谁在什么时候修改了什么 |
| 运维人员 | 作为运维人员，我希望对比任意两个版本的差异，以便理解配置变更的影响 |
| 运维人员 | 作为运维人员，我希望将配置回退到历史版本，以便在出问题后快速恢复 |
| 前台系统 | 作为前台开发者，我希望调用API获取/更新配置，以便系统能够读取最新配置 |
| 前台系统 | 作为前台开发者，我希望调用API查询历史版本，以便展示给用户 |

## 3. 功能列表

### F1: 配置项管理

- **F1.1** 配置列表：分页展示所有配置项（config_key, config_name, updated_at）
- **F1.2** 配置详情：查看单个配置的完整信息（包含当前JSON值）
- **F1.3** 新增配置：创建新的配置项
- **F1.4** 更新配置：修改配置值（自动记录历史）
- **F1.5** 删除配置：软删除配置项

### F2: 历史记录

- **F2.1** 历史列表：查看某配置项的所有变更历史（分页）
- **F2.2** 版本详情：查看某历史版本的完整快照
- **F2.3** 变更原因：记录每次变更的原因（可选）

### F3: 版本对比

- **F3.1** 差异对比：对比两个版本的JSON差异，高亮显示新增/修改/删除的字段

### F4: 版本回退

- **F4.1** 回退操作：将配置恢复到指定历史版本
- **F4.2** 回退记录：回退操作本身也生成新版本，保留完整审计链

### F5: API接口

- **F5.1** 完整RESTful API供前台调用
- **F5.2** 支持JSON格式的导入导出

## 4. 非功能需求

| 需求 | 描述 |
|------|------|
| 性能 | 历史记录查询支持分页，单次查询不超过100ms |
| 扩展性 | 支持扩展多个配置分类 |
| 审计 | 所有变更操作必须可追溯 |
| 兼容性 | API版本化（v1），支持未来升级 |

## 5. 数据模型

### 5.1 配置项表 (config_item)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| config_key | VARCHAR(100) | 配置键（唯一） |
| config_name | VARCHAR(200) | 配置名称 |
| config_value | JSON | 配置值（当前生效版本） |
| description | VARCHAR(500) | 描述 |
| deleted | TINYINT | 软删除标记 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 5.2 配置历史表 (config_history)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| config_id | BIGINT | 关联配置项ID |
| version_no | INT | 版本号（自增） |
| config_value | JSON | 配置快照 |
| change_type | VARCHAR(20) | 变更类型：UPDATE/ROLLBACK/INIT |
| operator | VARCHAR(100) | 操作人 |
| operator_ip | VARCHAR(50) | 操作人IP |
| change_reason | VARCHAR(500) | 变更原因 |
| created_at | DATETIME | 变更时间 |

## 6. API设计

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/configs | 获取配置列表（分页） |
| GET | /api/v1/configs/{id} | 获取配置详情 |
| POST | /api/v1/configs | 新增配置 |
| PUT | /api/v1/configs/{id} | 更新配置 |
| DELETE | /api/v1/configs/{id} | 删除配置 |
| GET | /api/v1/configs/{id}/history | 获取历史记录列表 |
| GET | /api/v1/configs/{id}/history/{versionId} | 获取指定版本详情 |
| POST | /api/v1/configs/{id}/rollback/{versionId} | 回退到指定版本 |
| GET | /api/v1/configs/{id}/diff | 对比两个版本 |
